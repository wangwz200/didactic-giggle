# 数据库结构设计说明书 v3.0

> 本文档用纯中文说明软件需要存哪些数据、怎么存，给项目相关人员阅读，不含具体代码实现。
> v2.0 新增：字段类型、约束说明、外键关系、索引定义、命名规范。
> v3.0 新增：学生端 / 教师端 / 管理员端 三端角色体系，课程管理、作业提交、成绩管理、审计日志等。

---

## 一、命名规范

| 规范项 | 规则 | 示例 |
|--------|------|------|
| 表名 | 小写 + 下划线，使用复数 | `users`, `course_enrollments`, `audit_logs` |
| 字段名 | 小写 + 下划线 | `created_at`, `parent_id` |
| 主键 | 统一叫 `id` | `id BIGINT AUTO_INCREMENT` |
| 外键 | 引用表名 + `_id` | `user_id`, `course_id` |
| 软删除标记 | 统一叫 `deleted_at` | `deleted_at DATETIME NULL` |
| 时间戳 | `created_at`, `updated_at` | 由程序自动填写 |
| 索引命名 | `idx_表名_字段名` | `idx_notes_user_id` |
| 唯一索引 | `uk_表名_字段名` | `uk_users_email` |

## 二、软件需要存哪些数据？

按功能 + 角色来分，一共需要记录 **14 类数据**：

| 功能 | 要存的内容 | 所属角色 |
|------|-----------|---------|
| 用户账号 | 注册邮箱、密码、昵称、头像、个人设置 | 共用 |
| 笔记本 | 分类文件夹，支持多级目录（无限层级） | 学生 / 教师 |
| 笔记 | 笔记标题、正文、标签、所属笔记本 | 学生 / 教师 |
| 任务清单 | 待办事项、优先级、截止日期、完成状态、子任务 | 学生 / 教师 |
| 番茄钟 | 每次专注的时长、开始结束时间、关联的任务 | 学生 / 教师 |
| 每日打卡 | 打卡日期、心情、学习时长 | 学生 / 教师 |
| 学习资料 | 上传的文件名、类型、大小、所属课程 | 学生 / 教师 |
| 学习统计 | 每天的总专注时间、完成任务数、笔记数 | 学生 / 教师 |
| **课程管理** | **课程名称、描述、教师、上课时间、学期** | **教师端** |
| **选课管理** | **学生选课关系、选课时间、状态** | **学生端 + 教师端** |
| **作业管理** | **作业标题、内容、截止日期、满分、关联课程** | **教师端** |
| **作业提交** | **学生提交的作业内容/文件、提交时间、分数** | **学生端 + 教师端** |
| **公告管理** | **标题、内容、发布范围、置顶、有效期** | **教师端 + 管理员端** |
| **系统审计日志** | **操作人、操作类型、IP、操作详情** | **管理员端** |

### 3.1 用户账号（users）— 三端共用

存用户注册时填写的信息和系统自动记录的信息。**v3.0 新增 `role` 字段区分三端**。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 用户编号，系统自动分配 |
| email | VARCHAR(255) | NOT NULL, UNIQUE | 邮箱，用来登录，不能重复 |
| phone | VARCHAR(20) | NULL, UNIQUE | 手机号，可填可不填 |
| password_hash | VARCHAR(255) | NOT NULL | 密码（bcrypt 加密，不会存明文） |
| nickname | VARCHAR(100) | NOT NULL | 昵称 |
| avatar_url | VARCHAR(500) | NULL | 头像图片地址 |
| bio | VARCHAR(500) | NULL | 个人简介 |
| **role** | **ENUM('student','teacher','admin')** | **NOT NULL, DEFAULT 'student'** | **角色：学生 / 教师 / 管理员** |
| **student_id** | **VARCHAR(50)** | **NULL, UNIQUE** | **学号（仅学生）** |
| **teacher_title** | **VARCHAR(50)** | **NULL** | **职称（仅教师，如教授/副教授）** |
| **department** | **VARCHAR(200)** | **NULL** | **院系/部门** |
| settings | JSON | NULL | 个人设置（主题色、番茄钟默认时长等） |
| last_login_at | DATETIME | NULL | 最后登录时间 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| updated_at | DATETIME | NOT NULL, ON UPDATE CURRENT_TIMESTAMP | 信息更新时间 |
| deleted_at | DATETIME | NULL | 注销时间（软删除标记） |

**索引**：
- `uk_users_email` — UNIQUE on `email`
- `uk_users_phone` — UNIQUE on `phone`
- `uk_users_student_id` — UNIQUE on `student_id`
- `idx_users_role` — on `role` — 按角色筛选用户
- `idx_users_department` — on `department`
- `idx_users_deleted_at` — on `deleted_at`

**三端说明**：
```
学生端  (role='student')  : 记笔记、做任务、番茄钟、打卡、选课、交作业
教师端  (role='teacher')  : 以上所有 + 创建课程、布置作业、批改打分、发公告
管理员端 (role='admin')    : 用户管理、系统配置、审计日志、全局公告
```

### ⭐ 3.9 课程管理（courses）— 教师端 新增

教师创建的课程，学生选课后可以查看。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 课程编号 |
| teacher_id | BIGINT | NOT NULL, FK -> users.id | 授课教师（role='teacher'） |
| name | VARCHAR(200) | NOT NULL | 课程名称 |
| description | TEXT | NULL | 课程简介 |
| cover_image_url | VARCHAR(500) | NULL | 课程封面图 |
| semester | VARCHAR(50) | NOT NULL | 学期，如"2025-2026-1" |
| department | VARCHAR(200) | NULL | 开课院系 |
| class_hours | INT | NULL | 总学时 |
| credits | DECIMAL(3,1) | NULL | 学分，如 3.0 |
| max_students | INT | NULL, DEFAULT 0 | 容量上限，0 表示不限 |
| status | ENUM('draft','published','archived') | NOT NULL, DEFAULT 'draft' | 课程状态 |
| created_at | DATETIME | NOT NULL | 创建时间 |

### ⭐ 3.10 选课管理（course_enrollments）— 学生端 + 教师端 新增

记录学生选择了哪些课程。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 选课编号 |
| course_id | BIGINT | NOT NULL, FK -> courses.id | 关联课程 |
| student_id | BIGINT | NOT NULL, FK -> users.id | 选课学生（role='student'） |
| status | ENUM('pending','approved','rejected','dropped') | NOT NULL, DEFAULT 'pending' | 选课状态 |
| enrolled_at | DATETIME | NOT NULL | 选课时间 |
| dropped_at | DATETIME | NULL | 退课时间 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 修改时间 |

**索引**：
- `uk_enrollments_course_student` — UNIQUE on `(course_id, student_id)` — 一个学生不能重复选同一门课
- `idx_enrollments_course_id` — on `course_id` — 查某课程的所有学生
- `idx_enrollments_student_id` — on `student_id` — 查某学生选的所有课程
- `idx_enrollments_status` — on `(course_id, status)` — 按选课状态筛选

**使用例子**：
```
学生：浏览课程列表 → 选择"高等数学" → 点击选课 → 等待教师审核/自动通过
教师：课程管理 → 选课名单 → 查看/审核选课申请
```

| **type** | **ENUM('assignment','course')** | **NOT NULL** | **作业成绩 / 最终课程成绩** |
| assignment_id | BIGINT | NULL, FK -> assignments.id | 关联作业（type='assignment' 时） |
| course_id | BIGINT | NOT NULL, FK -> courses.id | 所属课程 |
| student_id | BIGINT | NOT NULL, FK -> users.id | 学生 |
| submission_id | BIGINT | NULL, FK -> submissions.id | 关联提交记录（type='assignment' 时） |
| score | DECIMAL(5,1) | NULL | 得分 |
| comment | TEXT | NULL | 教师评语 |
| graded_by | BIGINT | NULL, FK -> users.id | 批改教师 |
| graded_at | DATETIME | NULL | 批改时间 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 修改时间 |

**索引**：
- `uk_grades_assign_student` — UNIQUE on `(assignment_id, student_id)` — 一个学生一个作业只有一个成绩
- `idx_grades_course_id` — on `course_id` — 查某课程的所有成绩
- `idx_grades_student_id` — on `student_id` — 查某学生的所有成绩
- `idx_grades_course_student` — on `(course_id, student_id)` — 查某学生在某课程的所有成绩

**成绩计算逻辑**：
```
最终课程成绩 = Σ(每次作业得分 × 对应权重)   (当 type='course' 时，score 为汇总结果)
```

**使用例子**：
```
教师：批改作业 → 打分 85 → 写评语"推导过程清晰" → 系统自动更新该生课程总评
学生：查看作业 → 看到分数和评语 → 查看课程总成绩单
```

## 六、数据保留多长时间

| 数据类型 | 保留策略 | 技术实现 |
|---------|---------|---------|
| 番茄钟记录 | 永久保留，不删除 | 无 deleted_at 字段 |
| 每日打卡 | 永久保留，不删除 | 无 deleted_at 字段 |
| 学习统计 | 永久保留，不删除 | 无 deleted_at 字段 |
| 课程 / 作业 | 不会真正删除（软删除） | deleted_at 标记 |
| 选课记录 | 退课后仍保留记录 | status='dropped' 标记 |
| 成绩 | 永久保留，不删除 | 无 deleted_at 字段 |
| 审计日志 | 永久保留，不删除 | 无 deleted_at 字段 |
| 上传的文件 | 用户删除后，再保留 30 天 | 定时任务清理 deleted_at < NOW() - 30d |
| 笔记 | 用户删除后，再保留 60 天 | 定时任务清理 deleted_at < NOW() - 60d |
| 已完成的任务 | 完成后 30 天自动归档 | 标记 status='archived' |

**软删除实现方式**：
- 查询时统一加 `WHERE deleted_at IS NULL` 条件
- 也可以创建视图（VIEW）自动过滤已删除记录

## 七、索引策略与查询优化

### 7.1 核心查询场景及对应索引

| 查询场景 | 所用索引 | 所属端 |
|---------|---------|:------:|
| 用户登录（邮箱/手机） | `uk_users_email`, `uk_users_phone` | 共用 |
| 按角色筛选用户 | `idx_users_role` | 管理员 |
| 查某课程的所有学生 | `idx_enrollments_course_id` | 教师 |
| 查某学生的所有课程 | `idx_enrollments_student_id` | 学生 |
| 查某课程的所有作业 | `idx_assignments_course_id` | 教师/学生 |
| 查某学生的所有成绩 | `idx_grades_student_id` | 学生 |
| 查某课程的所有成绩 | `idx_grades_course_id` | 教师 |
| 查某作业的所有提交 | `idx_submissions_assignment_id` | 教师 |
| 按学期查课程 | `idx_courses_semester` | 共用 |
| 审计日志时间范围查询 | `idx_audit_logs_created_at` | 管理员 |
| 按操作类型查日志 | `idx_audit_logs_action` | 管理员 |
| 有效公告查询 | `idx_announcements_active` | 共用 |
| 搜索笔记内容 | `FULLTEXT idx_notes_search` | 学生/教师 |

### 7.2 索引使用原则

1. **复合索引最左前缀**：查询条件按索引定义的字段顺序从左到右匹配
2. **避免索引冗余**：`idx_a` 和 `idx_a_b` 可以共存
3. **定期分析慢查询**：用 `EXPLAIN` 分析查询计划
4. **数据量较小时不要加太多索引**：小表（<1000 行）全表扫描更快
5. **JSON 字段索引**：使用虚拟生成列 + 索引，而不是直接索引 JSON

## 八、角色权限控制建议（RBAC）

如果需要更细粒度的权限控制，建议引入独立的 RBAC 表：

```
users (1) ────< (N) user_roles (N) >──── roles (1) ────< (N) role_permissions (N) >──── permissions
```

但当前三端场景（student / teacher / admin）使用 `users.role` ENUM 字段即可满足需求，无需复杂的 RBAC 系统。后续如果需要"课程助教"等细粒度权限，再引入 RBAC。

### 各角色默认权限范围

| 角色 | 可访问数据范围 | 可执行操作 |
|------|--------------|-----------|
| student | 自己的数据 + 已选课程的数据 | 增删改查自己的内容，提交作业，查看成绩 |
| teacher | 自己的数据 + 自己课程的数据 | 教师全部功能 + 管理课程内学生 |
| admin | 全系统数据 | 所有权限，包括用户管理、系统配置、日志审计 |

## 九、开发时需要遵守的规则

1. **软删除**：删除数据不要直接 DELETE，而是设置 `deleted_at` 时间戳
2. **时间戳自动管理**：`created_at` 和 `updated_at` 由程序自动填写，不手动赋值
3. **角色校验**：所有前端+后端接口必须校验用户角色，防止越权操作
4. **数据库迁移**：后续加新字段通过迁移脚本（`database/migrations/`）逐步更新
5. **迁移脚本规范**：
   - 文件名格式：`YYYYMMDDHHMMSS_description.sql`
   - 包含 `UP`（升级）和 `DOWN`（回滚）两部分
   - 示例：`20240601000001_create_users_table.sql`
6. **环境一致性**：开发阶段用 Docker 运行数据库
7. **查询安全**：
   - 所有用户输入必须参数化查询，防止 SQL 注入
   - 敏感数据（如邮箱、手机号）传输和存储需加密
8. **字符集**：统一使用 `utf8mb4` 字符集，`utf8mb4_unicode_ci` 排序规则
